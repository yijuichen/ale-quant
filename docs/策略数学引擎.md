# 策略数学引擎

**文档目标：** 本文档是策略逻辑的**数学规格书**，定义 `Step()` 函数的完整输入输出契约，以及内部各层（市场感知 / 宏观引擎 / 微观引擎 / 底仓释放）的计算逻辑。系统拓扑见《系统总体拓扑结构》，进化机制见《进化计算引擎》。

> **关于"设计空间"标记**：标注 **【设计决策 · 可调整】** 的章节是策略设计空间，下面给出的是一套内部自洽的具体方案与默认值，可整章替换。其余章节（资产三态架构、Sigmoid 公式、铁律）为通用架构，应保留。

---

## 第 0 章：引擎身份

策略引擎是一个**纯函数无状态转换器**。唯一外部入口：

```
Step(StrategyInput) → StrategyOutput
```

引擎不持有任何跨调用的内存状态——所有"记忆"都通过 `StrategyInput.RuntimeState` 传入、通过 `StrategyOutput.RuntimeState` 传出，由 SaaS 侧持久化。这保证了任意 tick 都可从数据库快照重建，且回测与实盘逐帧等价。

### 0.1 输入契约 `StrategyInput`

| 字段 | 类型 | 语义 |
|------|------|------|
| `Closes` | `[]float64` | 收盘价序列（已由 ACL 外圈从 OHLCV 降级，铁律 4） |
| `Timestamps` | `[]int64` | 与 `Closes` 对齐的毫秒时间戳序列 |
| `NowMs` | `int64` | 当前决策 tick 的时间戳（= 最新已完成聚合桶） |
| `Portfolio` | `PortfolioState` | 真实资产三态快照（见第 1 章） |
| `Runtime` | `RuntimeState` | 策略内部运行状态快照（上次 Step 产出） |
| `Params` | `Chromosome` | 当前激活的可进化参数（见第 7 章） |
| `Spawn` | `SpawnPoint` | Epoch 级冻结的出生点参数（见第 7 章） |
| `LotStep` | `float64` | 交易所下单数量精度 |
| `LotMin` | `float64` | 交易所最小下单数量 |

### 0.2 输出契约 `StrategyOutput`

| 字段 | 类型 | 语义 |
|------|------|------|
| `Intents` | `[]Intent` | 本 tick 的交易意图列表（可为空） |
| `Runtime` | `RuntimeState` | 更新后的运行状态，回写 DB |
| `Audit` | `[]AuditEntry` | 决策审计（含 reason code），用于事后重放 |

`Intent` 字段：`Engine`（`MACRO`/`MICRO`）、`Action`（`BUY`/`SELL`）、`AmountUSDT`（买入用）、`QtyAsset`（卖出用）、`LotType`（`DEAD_STACK`/`FLOATING`）、`ReasonCode`。`dead_stack → floating` 的释放（第 6 章）以**仅账本意图**（不下发 Agent）形式输出，由 SaaS 按铁律 9 落账。

### 0.3 铁律复述

- **复利前置**：引擎所有规模决策均以"占权益比例"表达，资金随权益滚动放大（见第 4 章宏观、第 1 章 SpendableUSDT）。
- **策略同构**：回测与实盘共用本 `Step()`，内部禁止 `if isBacktest` 分支。
- **策略纯函数**：禁止定时器、网络、数据库、文件 I/O；只依赖入参。
- **无量纲计算**：所有市场特征与信号均用对数收益率或比率表达，禁止裸价格跨标的比较。
- **仓位三态**：`DeadStack` / `FloatStack` / `ColdSealedStack` 语义不可混用。

## 第 1 章：资产结构三态（Portfolio State）

资产以三态区分语义，物理上是同一种标的资产，逻辑上账本分离：

| 态 | 语义 | 可否卖出 |
|----|------|---------|
| **DeadBTC** | 宏观底仓，长期趋势吸入 | **只进不出**，仅可经第 6 章释放规则转为 FloatBTC |
| **FloatBTC** | 微观浮动仓，Sigmoid 动态天平调仓对象 | 可买可卖 |
| **ColdSealedBTC** | 冷封存仓，出生点冻结的保命底仓 | **永不释放、永不卖出** |

### 1.1 通用公式（保留，不属设计空间）

```
TotalEquity      = (DeadBTC + FloatBTC + ColdSealedBTC) × Price + USDT_total

ReserveFloor     = micro_reserve_pct × TotalEquity

SpendableUSDT    = max(0, USDT_available − ReserveFloor)

CurrentMicroWeight = FloatBTC × Price / TotalEquity
```

- `TotalEquity`：全口径净值，三态资产 + 全部 USDT，按当前价折算。复利的滚动基数。
- `ReserveFloor`：必须保留、不可被微观引擎动用的 USDT 下限（按权益比例，随权益增长而抬升，保证永远留有干火药）。
- `SpendableUSDT`：微观与宏观引擎可动用的现金，扣除保留底线后的余额。
- `CurrentMicroWeight`：浮动仓占总权益比重，即 Sigmoid 动态天平的 `CurrentWeight`（第 5 章直接引用）。

### 1.2 `micro_reserve_pct` 参数

| 名称 | 语义 | 默认值 | 是否可进化 |
|------|------|--------|-----------|
| `micro_reserve_pct` | 保留 USDT 占权益比例 | `0.05`（5%） | 是（染色体，见第 7 章） |

## 第 2 章：市场状态感知层　【设计决策 · 可调整】

> **设计空间一：如何对市场状态分类？** 下面方案将市场分为**三态**，全部用无量纲特征判断。安静态下微观粉尘订单强制归零。

### 2.1 状态枚举

| 状态 | 含义 | 对系统的影响 |
|------|------|-------------|
| `BULL`（牛） | 趋势向上 | 放大微观 β（更激进调仓），宏观加速吸入 |
| `BEAR`（熊） | 趋势向下 | 放大微观 β，宏观逢低加码（见第 4 章） |
| `QUIET`（安静） | 无趋势、低波动 | **微观粉尘订单归零**，β 不放大，减少无效交易 |

### 2.2 判断特征（均无量纲）

```
TrendScore     = ln(EMA_fast / EMA_slow)              // 趋势方向与强度，正=向上
VolatilityRatio = clip(MAV_short / MAV_long, 0.1, 3.0) // 波动率比值（MAV=平均绝对涨跌）
```

`MAV`（Mean Absolute Variation，平均绝对涨跌）为窗口内逐根对数收益绝对值的均值，非 ATR。

### 2.3 分类逻辑（优先级自上而下）

```
若 |TrendScore| < trend_eps 且 VolatilityRatio < quiet_vol_threshold:
        State = QUIET
否则若 TrendScore ≥ +trend_eps:
        State = BULL
否则若 TrendScore ≤ −trend_eps:
        State = BEAR
否则:
        State = QUIET   // 有波动但无明确趋势，仍按安静处理
```

### 2.4 输出：`MarketBetaMultiplier`

市场感知层向微观引擎输出 β 放大系数，行情越极端放大越多：

```
State = QUIET → MarketBetaMultiplier = 1.0
State = BULL  → MarketBetaMultiplier = clamp(1 + bull_beta_gain × |TrendScore|, 1.0, beta_mult_max)
State = BEAR  → MarketBetaMultiplier = clamp(1 + bear_beta_gain × |TrendScore|, 1.0, beta_mult_max)
```

`QUIET` 态下，第 5 章楔形过滤的"非安静态"条件不成立，粉尘订单一律归零。

## 第 3 章：信号与目标函数框架（可选 MPC 概念层）

微观引擎的 `Signal` 是多个无量纲特征的线性合成（第 5 章），本质是一个**单步控制律**：根据当前市场特征直接解出目标持仓权重。

可选的概念升级路径（**当前不实现，仅记录设计意图**）：将单步控制律推广为**模型预测控制（MPC）**——在每个 tick 用冻结的价格模型预测未来 N 步，在预测窗口上最小化"偏离目标权重 + 交易成本"的代价泛函，取首步动作执行，下一 tick 重新求解（滚动时域）。轨迹冻结类平仓模型（见《量化交易平仓策略》）即 MPC 思想在平仓维度的一个具体落地。本文档第 5 章只规定单步律；MPC 留作未来扩展，不进入当前 `Step()`。

## 第 4 章：宏观引擎（趋势 DCA）　【设计决策 · 可调整】

> **设计空间二：宏观引擎如何决定何时买、买多少？** 宏观引擎关注**长期趋势**，只买不卖（吸入 DeadBTC），用"价值定投 + 逢低加码"实现复利累积。

### 4.1 触发节奏

宏观引擎按 `macro_period_days` 的自然周期触发一次吸入（默认 30 天，对齐自然月边界）。`RuntimeState` 记录 `LastMacroBuyMs`，距今不足一个周期则本 tick 不出宏观意图。这与微观引擎（每 tick 评估）解耦——宏观慢、微观快。

### 4.2 逢低信号（无量纲）

```
DipRatio = ln(EMA_macro_long / Price)     // 价格低于长期均线时为正（处于价值区）
```

`EMA_macro_long` 为长周期均线（默认 200 日等效），代表长期趋势中枢。

### 4.3 买入规模（复利前置：以权益比例表达）

```
MacroBudget = macro_base_pct × TotalEquity
            × clamp(1 + macro_dip_gain × DipRatio, macro_min_mult, macro_max_mult)

ActualBuyUSDT = min(MacroBudget, SpendableUSDT)
```

- 基准额 `macro_base_pct × TotalEquity` 随权益滚动 → **复利成立**。
- 价格越低于长期中枢（`DipRatio` 越大），乘子越大 → 逢低多买；价格高于中枢则乘子收敛到 `macro_min_mult` → 高位少买甚至最低额。
- 受 `SpendableUSDT` 上限约束，绝不动用 `ReserveFloor`。

### 4.4 产出

若 `ActualBuyUSDT ≥ LotMin 对应金额`，产出一条 `Intent{Engine:MACRO, Action:BUY, AmountUSDT:ActualBuyUSDT, LotType:DEAD_STACK}`，并更新 `RuntimeState.LastMacroBuyMs`。宏观引擎**永不产出 SELL**（只进不出，卖出只能经第 6 章释放后由微观执行）。

## 第 5 章：微观引擎（Sigmoid 动态天平）

直接采用 Sigmoid 动态天平。微观引擎用 Sigmoid 函数实时计算 `FloatBTC` 的**目标持仓权重**，通过买卖浮动仓使实际权重趋近目标。

### 5.1 信号合成

```
Signal = a × X1 + b × X2 + c × X3 + ...
```

X1/X2/X3 为你选择的无量纲市场特征（如价格偏离类、动量类、加速度/突破类），a/b/c 为权重系数（**染色体，由 GA 搜索**，见第 7 章）。`Signal` 正值倾向减仓、负值倾向加仓。

### 5.2 Sigmoid 目标权重

```
CurrentWeight  = FloatBTC × Price / TotalEquity          // = 第 1 章 CurrentMicroWeight
EffectiveBeta  = max(0.01, β × MarketBetaMultiplier)     // β 来自第 7 章，乘子来自第 2 章
InventoryBias  = clamp(CurrentWeight, 0, 1) − 0.5
Exponent       = EffectiveBeta × Signal + γ × InventoryBias
TargetWeight   = 1 / (1 + e^Exponent),  clamp(0, 1)
```

- **β（激进系数）**：越大调仓越频繁；市场感知层在极端行情放大 β。
- **γ（仓位偏置系数）**：`γ=0` 纯信号驱动；`γ>0` 叠加均值回归力，防止仓位极端漂移。

### 5.3 理论订单

```
DeltaWeight    = TargetWeight − CurrentWeight
TheoreticalUSD = DeltaWeight × TotalEquity
```

`TheoreticalUSD > 0` → 加仓（BUY，受 `SpendableUSDT` 约束）；`< 0` → 减仓（SELL，卖出 `FloatBTC`）。

### 5.4 楔形区过滤（粉尘拦截）

```
若 |TheoreticalUSD| ≥ min_order_usd:
        直接下单
若 |TheoreticalUSD| ∈ (0, min_order_usd):
        仅在「非安静态（State ≠ QUIET）」且满足楔形突破条件时，强制最小订单；否则归零
```

楔形突破条件：`|DeltaWeight| ≥ delta_weight_threshold` **或** `VolatilityRatio ≥ vol_ratio_threshold`。其中 `VolatilityRatio = clip(MAV_short / MAV_long, 0.1, 3.0)`（与第 2 章同源）。

## 第 6 章：DeadBTC 释放规则　【设计决策 · 可调整】

> **何时可把宏观买入的 DeadBTC 转为浮动仓（即变为可卖出）？** 释放仅在 SaaS 侧改账本（`dead_stack → floating`），**不向 Agent 下发任何 TradeCommand**，但必须写审计日志（铁律 9）。`ColdSealedBTC` 永不参与释放。

### 6.1 释放信号

当价格显著高于长期中枢（已脱离价值区、进入高位），释放部分底仓给微观引擎，使其有筹码在高位逐步分发（卖高）：

```
PremiumRatio = ln(Price / EMA_macro_long)     // 价格高于长期均线时为正
```

### 6.2 释放条件与额度

```
若 PremiumRatio ≥ release_threshold  且  距上次释放 ≥ release_period_days:
        ReleaseQty = release_fraction × DeadBTC
        DeadBTC  -= ReleaseQty
        FloatBTC += ReleaseQty
        写审计（reason = dead_stack_release），更新 RuntimeState.LastReleaseMs
```

- 高位才释放 → 与"宏观逢低买、微观高位卖"形成完整的低买高卖闭环。
- 按比例分批释放（`release_fraction`），受 `release_period_days` 限频，避免一次性倾倒。
- 释放只改三态账本归属，不产生真实成交；后续是否卖出由微观 Sigmoid 在浮动仓上自行决策。

## 第 7 章：可进化参数契约（Chromosome）　【设计决策 · 可调整】

> **设计空间三：哪些参数交给 GA 搜索？** 分两类：**染色体**（参与代内交叉/变异，进入基因组指纹）与**出生点参数**（Epoch 级冻结，全种群共享，不进入基因组）。区分依据见《进化计算引擎》第 1.3 节。

### 7.1 染色体字段清单（参与代内交叉变异）

| 字段 | 类型 | 语义 | 默认值 | 硬边界 |
|------|------|------|--------|--------|
| `micro_reserve_pct` | float | 保留 USDT 占权益比例 | 0.05 | [0.0, 0.30] |
| `beta` (β) | float | 微观激进系数 | 2.0 | [0.1, 10.0] |
| `gamma` (γ) | float | 仓位偏置系数 | 1.0 | [0.0, 5.0] |
| `sig_coef_a` | float | 信号特征 X1 权重 | 1.0 | [−5.0, 5.0] |
| `sig_coef_b` | float | 信号特征 X2 权重 | 0.0 | [−5.0, 5.0] |
| `sig_coef_c` | float | 信号特征 X3 权重 | 0.0 | [−5.0, 5.0] |
| `ema_fast_period` | int | 快速 EMA 周期 | 12 | [2, 100] |
| `ema_slow_period` | int | 慢速 EMA 周期 | 26 | [5, 300] |
| `ema_macro_long_period` | int | 长期中枢 EMA 周期 | 200 | [50, 1000] |
| `mav_short_period` | int | 短期 MAV 周期 | 14 | [2, 100] |
| `mav_long_period` | int | 长期 MAV 周期 | 60 | [10, 400] |
| `trend_eps` | float | 趋势/安静阈值 | 0.01 | [0.0, 0.20] |
| `quiet_vol_threshold` | float | 安静态波动率上限 | 0.8 | [0.1, 2.0] |
| `bull_beta_gain` | float | 牛市 β 放大增益 | 5.0 | [0.0, 50.0] |
| `bear_beta_gain` | float | 熊市 β 放大增益 | 5.0 | [0.0, 50.0] |
| `beta_mult_max` | float | β 放大上限 | 3.0 | [1.0, 10.0] |
| `macro_period_days` | int | 宏观吸入周期 | 30 | [1, 365] |
| `macro_base_pct` | float | 宏观基准额占权益比 | 0.02 | [0.0, 0.20] |
| `macro_dip_gain` | float | 逢低加码增益 | 3.0 | [0.0, 30.0] |
| `macro_min_mult` | float | 宏观买入乘子下限 | 0.3 | [0.0, 1.0] |
| `macro_max_mult` | float | 宏观买入乘子上限 | 4.0 | [1.0, 10.0] |
| `release_threshold` | float | DeadBTC 释放溢价阈值 | 0.30 | [0.05, 1.0] |
| `release_fraction` | float | 单次释放比例 | 0.10 | [0.01, 0.50] |
| `release_period_days` | int | 释放限频 | 30 | [1, 365] |
| `min_order_usd` | float | 粉尘最小订单阈值（USD） | 10.0 | [1.0, 1000.0] |
| `delta_weight_threshold` | float | 楔形权重突破阈值 | 0.05 | [0.0, 0.5] |
| `vol_ratio_threshold` | float | 楔形波动率突破阈值 | 1.5 | [0.1, 3.0] |

### 7.2 出生点参数（Epoch 级冻结，不进入基因组）

对齐《进化计算引擎》的 `SpawnPoint`（`CapitalPolicy` + `RiskBounds`），定义资金与风险环境，全种群共享、代内不变：

| 字段 | 归属 | 语义 | 默认值 |
|------|------|------|--------|
| `seed_capital` | CapitalPolicy | 种子资本（USDT） | 由任务指定 |
| `monthly_inject` | CapitalPolicy | 月度注资额（与 Ghost DCA 基准对齐） | 由任务指定 |
| `cold_sealed_pct` | RiskBounds | 出生即封存的冷仓比例（永不释放） | 0.10 |

> 出生点为何不进基因组：它定义的是"评估环境"而非"策略行为"。同一套策略基因应在固定的资金/风险环境下被公平比较；若让资金环境随个体变异，适应度将不可比。

### 7.3 硬边界约束

所有字段越界时由策略的 `Clamp` 修复到上表硬边界内（交叉/变异后调用）。`int` 型周期字段修复后取整。

### 7.4 结构约束

交叉变异可能破坏字段间的相对关系，`Clamp` 必须额外修复以下结构约束：

1. **EMA 顺序约束（时间周期相对论锁）**：`ema_fast_period < ema_slow_period < ema_macro_long_period`。修复策略：越界时按比例顶开或交换，保证严格升序。
2. **MAV 顺序约束**：`mav_short_period < mav_long_period`。
3. **宏观乘子约束**：`macro_min_mult ≤ 1.0 ≤ macro_max_mult`。
4. **β 放大约束**：`beta_mult_max ≥ 1.0`。

任一结构约束修复后，基因方为合法（`Validate` 通过），方可进入适应度评估。
