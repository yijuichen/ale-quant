# 系统总体拓扑结构

**文档目标：** 定义系统有哪些物理端、有哪些逻辑模块、状态如何在它们之间流转，以及系统的生命周期动作。本文档是系统拓扑的唯一真源，**不含任何具体策略公式**（策略数学交由《策略数学引擎》文档，进化机制交由《进化计算引擎》文档）。

---

## 第 0 章：架构哲学

**时空正交，无记忆解算。** 所有业务推进必须被封装为离散的"动作"，动作之间严禁隐式状态传递（禁止"欠债记录"）。每个动作读入当前状态快照，产出新的确定性意图，不依赖上一动作残留在内存里的变量。这保证了任意时刻系统都可以从数据库快照重建，断电、重启、断线都不会撕裂状态。

**窗口隔离原则。** 每个模块只能读取属于自己的"数据窗口"与"资金窗口"。策略模块看不到数据库内部结构，宏观引擎看不到微观浮仓的成交细节，进化引擎看不到实盘账户。模块之间只通过明确定义的契约（`StrategyInput` / `StrategyOutput` / `TradeCommand` / `DeltaReport`）通信。

**云端只信上报，端侧无脑执行。** 决策权完全收敛在云端 SaaS，端侧 Agent 不做任何自主判断。所有真实账户状态以 Agent 上报的余额快照为准，这使得网络断线、丢包、脏数据都能通过"下一次基于真实余额重新决策"自然自愈，无需补偿逻辑。

## 第 1 章：三端物理部署形态

系统由三个物理端构成，每端职责单一、边界清晰，靠 `app_role` 在路由层强制隔离能力。

### 1.1 SaaS 管理端（云端，`app_role: saas`）

**职责：** 决策大脑。对策略模板、实例、用户认证与订阅做全量管理。通过 cron 扫描 `RUNNING` 实例，仅在该实例最新已完成的聚合桶尚未处理时执行一次 `Step()`，将产出意图翻译为 `TradeCommand`，经 WebSocket Hub 下发给在线 Agent 并接收成交上报。

**禁区：** 不执行 GA 进化与回测运算（在路由层拦截写接口）；**永不保存任何交易所 API 凭证**。

### 1.2 LocalAgent 执行端（用户本地）

**职责：** 执行手。极简二进制进程，运行在用户本地。启动时通过 REST 登录获取 JWT，建立 WebSocket 长连到 `/ws/agent`；收到 `TradeCommand` 后调用交易所 REST 现货下单，获取成交明细与余额构建 `DeltaReport` 上报；定时心跳，断线指数退避自动重连。

**禁区：** 不含任何策略代码，不连接数据库，不自主决策。凭证仅来自本端 `config.agent.yaml`。

### 1.3 进化算力面（本地实验室，`app_role: lab`）

**职责：** 实验室。同样是完整的 SaaS 系统，但部署在本地算力机上，**连接云端同一个 Postgres 实例**，专注执行 GA 进化任务与回测运算，产出 challenger 基因包写回共享数据库。

**禁区：** 不下发真实交易指令（路由层禁用实例启停与交易下发，实例对 lab 只读）。

## 第 2 章：app_role 三态行为矩阵

同一份代码库通过启动配置中的 `app_role` 决定开放哪些能力，路由层据此守门，物理隔离决策、进化、实盘三类职责。

| 角色 | 部署场景 | 开放能力 | 限制 |
|------|---------|---------|------|
| `saas` | 云端 SaaS | 实例管理、`Step()` 决策、交易指令下发、用户运营 | 禁止进化/回测写接口 |
| `lab` | 本地实验室 | GA 进化、回测、基因库管理 | 禁止交易指令下发，实例只读 |
| `dev` | 开发测试 | 所有模块全开 | 无限制 |

## 第 3 章：逻辑模块与职责边界

### 3.1 Strategy 策略模块（产品中心）

管理策略模板（图纸）：参数结构、版本标识、交易对类型（现货/合约）。`Step()` 是策略决策的**唯一外部入口**，所有买卖意图、仓位规模、风控判断都在其内部产生；回测与实盘共用同一份 `Step()`，不允许内部分叉。**禁区：** 不读数据库、不发网络、不做文件 I/O。

### 3.2 Instance 实例模块（设备中心）

实例的本质 = 策略模板 + 交易标的 + 资金配额 + 用户授权。负责实例的创建/启动/停止/删除，以及运行态（`RUNNING` / `STOPPED` / `ERROR`）的维护。cron 按基础频率扫描 `RUNNING` 实例，聚合桶去重后推进 tick：拉公开 K 线、读 `PortfolioState`、构建 `StrategyInput`、调 `Step()`、翻译 `TradeCommand` 下发。**禁区：** 不侵入策略内部逻辑。

### 3.3 Evolution 进化模块（参数实验室）

通过 GA 对策略参数进化迭代，产出 challenger 参数包；人工前端审批后晋升为 champion，供实例绑定。回测路径调用与实盘相同的 `Step()`。**禁区：** 只在 `lab` / `dev` 开放，不接触实盘账户与凭证，不 import 任何具体策略实现（通过抽象接口驱动）。

### 3.4 Auth 认证模块

处理用户注册、登录、JWT 签发，以及订阅计划校验（创建实例、启动实例时做配额守门）。**禁区：** 不涉及任何策略逻辑，与决策、进化解耦。

## 第 4 章：全局状态总线

### 4.1 单一 Postgres + Redis 仅缓存

整个系统使用单一 Postgres 数据库，不分库（不区分 Shell DB / Live DB）。三端连接同一实例。Redis **仅作缓存**（冠军基因 key、会话等），绝不承担信号传递职责。数据库 schema 以 Go struct 为唯一真源，通过 GORM AutoMigrate 同步。

### 4.2 数据真源与所有权

每一类数据都有且只有一个真源端，任何其他端只能读取副本或经由真源端更新。

| 数据 | 真源 | 说明 |
|------|------|------|
| 策略决策逻辑 | `Step()` 实现（SaaS 侧执行） | 回测与实盘必须共用 |
| 冠军参数包 / 模板 | SaaS DB | 进化产出，供实例绑定 |
| 终端用户与订阅 | SaaS DB | 统一在 SaaS 管理 |
| 运行中实例状态 | SaaS DB | 由 Agent 上报后维护 |
| 交易所 API 凭证 | LocalAgent 本地配置文件 | 仅 Agent 持有，永不进入 SaaS |
| 成交记录与资产快照 | SaaS DB | Agent 上报后持久化 |
| GA 任务与基因记录 | SaaS DB | 单一数据库，无分库 |

### 4.3 交易所凭证物理隔离铁律

交易所 API Key / Secret / Passphrase **只能**存在于 LocalAgent 的 `config.agent.yaml`：永不进入 SaaS 服务、永不写入任何数据库表、永不通过网络传输到云端。发现代码将凭证写入 SaaS 侧时，必须立即停止并报告。

## 第 5 章：WebSocket 通信协议

云端与端侧严格遵循"云端只信上报，端侧无脑执行"机制。Agent 发起连接，SaaS 监听 `/ws/agent`；连接建立后 Agent 先发鉴权消息，验证通过后进入收发循环，每 30 秒心跳。

### 5.1 消息类型全表

| 消息类型 | 方向 | 触发时机 |
|---------|------|---------|
| `auth` | Agent → SaaS | 连接建立后立即发送，携带 JWT |
| `auth_result` | SaaS → Agent | 鉴权完成后返回 |
| `heartbeat` | Agent → SaaS | 每 30 秒 |
| `heartbeat_ack` | SaaS → Agent | 收到心跳后 |
| `command` | SaaS → Agent | `Step()` 产出意图后，翻译为具体指令下发 |
| `command_ack` | Agent → SaaS | 收到 command 后立即确认（不等执行完成） |
| `delta_report` | Agent → SaaS | 交易执行完成后；或重连时的初始余额快照 |
| `report_ack` | SaaS → Agent | SaaS 成功处理 DeltaReport 后返回 |

### 5.2 TradeCommand 字段语义

| 字段 | 取值范围 | 说明 |
|------|---------|------|
| `client_order_id` | 全局唯一字符串 | 格式 `inst{id}-{type}-{ts}`，用于去重与审计 |
| `action` | `BUY` / `SELL` | 交易方向（大写） |
| `engine` | `MACRO` / `MICRO` | 产生此指令的引擎层（大写） |
| `symbol` | 如 `[你的交易对]` | 交易对 |
| `amount_usdt` | 字符串金额 | 买入时使用，表示花费 USDT 数量 |
| `qty_asset` | 字符串数量 | 卖出时使用，表示卖出标的资产数量 |
| `lot_type` | `DEAD_STACK` / `FLOATING` | 仓位语义：宏观吸入底仓 vs 微观浮动仓 |

下发 `TradeCommand` 的同时在 DB 写一条 pending 记录，供 DeltaReport 回填时按 `lot_type` 拆分更新底仓 / 浮仓。

### 5.3 DeltaReport 字段语义

| 字段 | 说明 |
|------|------|
| `balances` | 当前所有资产的余额快照（标的资产与 USDT 的可用与冻结） |
| `execution` | 本次成交明细（成交量、成交价、手续费、状态） |
| `client_order_id` | 对应的指令 ID（重连发送初始快照时可为空） |

### 5.4 状态收敛与天然自愈

SaaS 收到 DeltaReport 后刷新对应实例的 `PortfolioState`，持久化成交记录与资产快照，写审计日志。下次 `Step()` 基于真实余额重新决策，由此天然自愈：Agent 断线时 SaaS 跳过该实例 tick，无须人工干预；Agent 重连立即发初始余额快照，下次 tick 自动纠偏；SaaS 重启后从 DB 恢复 `RUNNING` 实例列表，等待 Agent 重连。

## 第 6 章：系统级生命周期动作

### 6.1 系统初始化（System Init）

SaaS 进程启动时：① 读配置，初始化 DB / Redis / JWT 秘钥；② 执行 GORM AutoMigrate，以 Go struct 为唯一 schema 真源同步结构；③ 从 DB 读取所有 `RUNNING` 实例注册到内存管理列表；④ 启动 WebSocket Hub 开始接受 Agent 连接；⑤ 启动 cron 调度器注册实例基础扫描任务，具体 `Step()` 推进由各实例聚合桶去重控制。

### 6.2 Cron Tick 驱动 Step()

cron 基础扫描命中 `RUNNING` 实例后，若该实例最新已完成的聚合桶尚未处理则触发一次 `Step()`，已处理同一桶则跳过（避免实盘比回测多跑决策）。流程：① 拉公开 K 线；② 读 `PortfolioState`（上次 DeltaReport 维护的真实账户）；③ 读 `RuntimeState`（策略内部运行快照）；④ 构建 `StrategyInput`；⑤ 调 `Step()`（**只在 SaaS 侧执行**）；⑥ 翻译为带全局唯一 `client_order_id` 的 `TradeCommand`；⑦ DB 写 pending 记录；⑧ 经 Hub 下发对应 Agent。Agent 未连接则跳过、记警告、下次 tick 重试。

### 6.3 Agent 断线重连（指数退避）

Agent 检测到连接断开后：① 等待退避时间尝试重新登录获取 JWT；② 建立新 WebSocket 连接；③ 连接建立后立即发初始 `DeltaReport`（当前交易所余额快照，不含 `client_order_id` 与 `execution`）；④ SaaS 更新相关实例 `PortfolioState`；⑤ 下次 tick 基于真实余额重新决策。**退避策略：初始 1 秒，每次翻倍，最大等待 5 分钟。**

### 6.4 优雅停机与状态快照（Graceful Shutdown）

SaaS 收到 `SIGTERM` 等中断信号时：① 停止接受新连接与新 cron tick；② 等待执行中的 tick 完成（超时保护）；③ 持久化所有活跃实例的 `RuntimeState` 快照，避免重启后策略状态撕裂；④ 关闭所有 WebSocket 连接并发关闭帧；⑤ 关闭 DB 与 Redis 连接。

## 第 7 章：不可推翻的技术决策

以下为系统硬约束，违反时必须停止开发并报告，不得绕过。

1. **复利前置（铁律）**：任何策略机制的设计或实现，必须能清楚说明复利如何发生（资金规模随权益正反馈滚动）；不满足可论证复利性质的设计不予开发。
2. **策略同构**：回测与实盘必须调用同一个 `Step()` 实现，内部禁止出现 `if isBacktest` 分支；`Step()` 只在 SaaS 侧执行，Agent 二进制不含任何策略代码。
3. **策略纯函数**：策略代码内部绝对禁止定时器、网络请求、数据库读写、任何文件 I/O；只依赖传入的 `StrategyInput`，只输出 `StrategyOutput`。
4. **现货策略 ACL OHLC 剥离**：`IsSpot == true` 的策略，内核禁止依赖 OHLCV 聚合结构；OHLCV 到（收盘价序列 + 时间戳序列）的降级必须在 ACL 外圈（回测适配器入口与实盘 tick 入口）完成。
5. **API Key 物理隔离**：交易所凭证只允许存在于 LocalAgent 本地配置文件，永不进入 SaaS、永不写入数据库、永不经网络传输。
6. **GORM AutoMigrate 为唯一 schema 真源**：数据库结构以 Go struct 为真源，不写 SQL migration 文件，不维护版本化 migration 脚本。
7. **无量纲计算**：所有价格相关计算使用对数收益率或比率，禁止用绝对价格做跨标的比较。
8. **仓位三态**：资产以 `DeadStack`（宏观长期底仓）/ `FloatStack`（微观浮动仓）/ `ColdSealedStack`（冷封存，永不释放）三态区分语义，不可混用；更新规则由 `Step()` 决定，SaaS 按 `lot_type` 落账。
9. **底仓释放仅 SaaS 侧更新**：`dead_stack → floating` 的语义转换只改 SaaS 侧 `PortfolioState` 账本，不向 Agent 下发任何 `TradeCommand`，但必须写审计日志。
10. **单一数据库，无分库**：使用单一 Postgres 实例，不分 Shell/Live DB，不用 Redis 作信号传递通道（Redis 仅缓存）。
11. **用户界面零数学**：面向用户的字符串避免内部状态机术语、无上下文的裸数学量名或公式片段；希腊字母若必须保留须"字母 + 文字释义"并置（如「σ（标准差）」），禁止只展示符号而无说明。
