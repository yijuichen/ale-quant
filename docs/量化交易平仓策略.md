# 平仓逻辑产品需求文档（PRD）

**文档版本**：1.0  
**状态**：草案，供独立项目实现参考  
**适用范围**：具备策略决策、订单执行、回测与实盘双通道的自动化交易系统  

---

## 1. 文档目的

本文档定义**平仓子系统**的产品边界、业务规则、状态契约与验收标准，供产品、研发、测试在新项目中对齐实现。不涉及具体技术栈、仓库结构或既有系统命名。

---

## 2. 背景与问题陈述

### 2.1 业务背景

交易策略在持仓期间需要持续回答三类问题：

1. **何时止盈**：价格朝有利方向运动时，如何锁定利润而不过早离场。  
2. **何时止损**：价格朝不利方向运动时，如何在可接受代价下退出。  
3. **何时强制退出**：与模型无关的风控条件（超时、最小可交易规模、异常行情等）触发时如何全平。

若平仓逻辑与开仓逻辑耦合过紧，或回测与实盘采用两套规则，将导致绩效不可复现、实盘与仿真偏差不可解释。

### 2.2 本 PRD 要解决的问题

- 统一「平仓决策」的输入、输出与优先级，避免多处重复判断。  
- 支持**可切换的平仓模型**（至少一种默认模型 + 一种可选模型），切换不改变开仓与加仓规则。  
- 明确**全仓平仓（All-or-Nothing）**与**部分减仓**的产品策略（本 PRD 默认全仓）。  
- 保证回测与实盘在**同一套平仓决策函数**上运行，执行层仅负责下单与成交反馈。

## 3. 目标与非目标

### 3.1 目标

| 编号 | 目标 |
|------|------|
| G1 | 持仓生命周期内，每个行情 tick 输出唯一、可审计的平仓意图（持有 / 挂保护单 / 市价全平） |
| G2 | 平仓模型可配置、可持久化到任务/实例配置，缺省行为与历史数据兼容 |
| G3 | 加仓不重置已冻结的平仓参考轨迹（若采用轨迹类模型） |
| G4 | 用户界面展示平仓相关状态时使用业务语言，不暴露内部公式符号 |
| G5 | 回测对保护单触发的成交价规则与产品定义一致（见第 8 节） |

### 3.2 非目标

- 不定义开仓、选币、资金费率、杠杆调节等产品细节。  
- 不规定交易所 API、券商柜台或具体编程语言。  
- 不规定遗传算法、参数搜索或运维部署方式。  
- 不在本文档中固定「必须使用对数价格」或「必须使用三次多项式」；若采用，须在项目设计附录中单独论证。

## 4. 术语

| 术语 | 定义 |
|------|------|
| 持仓 | 策略认定的有效头寸，含方向（多/空）、开仓均价、开仓时间、名义规模 |
| 平仓模型 | 在持仓期间计算止盈/止损/强平条件的算法族，同一时刻仅激活一种 |
| 冻结锚点 | 自首次建仓起不再重算的参考量（如参考曲线、参考价带），加仓不改变锚点 |
| 保护单 | 交易所或执行层托管的限价/触发单，用于止盈或止损 |
| 市价全平 | 以市价（或等价指令）一次性平掉该标的全部持仓 |
| 贴合度 | 实际价格与模型预测价格的一致程度，取值 0–1，仅表示拟合好坏，不表示盈亏方向 |
| 有利偏离 | 价格相对预测朝**对持仓有利**的方向突破容忍带 |
| 不利偏离 | 价格相对预测朝**对持仓不利**的方向突破容忍带 |
| Tick | 策略决策的最小时间粒度（K 线收盘、定时轮询等由项目定义） |

---

## 5. 用户角色与场景

| 角色 | 场景 |
|------|------|
| 策略运营 | 为某交易实例选择平仓模型；查看当前保护价、贴合度、偏离方向 |
| 研发 | 实现平仓模型插件、状态持久化、与执行层协议对接 |
| 测试 | 按第 11 节验收用例做回测/仿真/界面回归 |
| 风控 | 配置全局强平规则（最大持仓时长、最小名义等），优先级高于模型内止损 |

## 6. 功能需求

### 6.1 平仓模型注册与切换

**FR-01** 系统应支持至少两种平仓模型标识，例如：

- `baseline`：项目定义的默认平仓逻辑（上线前必须实现）。  
- `trajectory_frozen`：开仓后冻结一条价格轨迹，后续 tick 仅基于「实际价 vs 冻结轨迹」决策（可选实现）。

**FR-02** 平仓模型为**运行配置**，不属于策略可进化参数集合；切换模型不触发重新开仓或重置 spawn/种子类配置。

**FR-03** 配置缺省、空值、历史未写字段时，一律按 `baseline` 处理，保证向后兼容。

**FR-04** 用户可在「任务创建 / 实例编辑」界面选择平仓模型；列表接口与详情接口均返回当前模型标识及可读名称。

### 6.2 决策流水线（每个 Tick）

**FR-05** 每个 tick 对**每个有效持仓**按固定优先级输出至多一种平仓动作：

```
优先级（高 → 低）：
1. 全局风控：dust（不可交易碎仓）、最大持仓时长、策略级熔断等
2. 模型级硬止损：不利偏离超过硬阈值 → 市价全平
3. 模型级保护单：满足止盈/保护止损条件 → 下发或更新保护单
4. 默认：继续持有（hold）
```

**FR-06** 同一 tick 内不得同时下发「市价全平」与「仅更新保护单」；硬止损触发时取消或覆盖未成交的保护单逻辑由执行层规范，但策略层只输出一种主意图。

**FR-07** 加仓（同向加码）后：

- 持仓均价、名义、盈亏展示可更新；  
- **冻结锚点不得因加仓而重算**（适用于 `trajectory_frozen`）；  
- 全仓平仓目标数量始终为当前总持仓量。

### 6.3 轨迹冻结类模型（`trajectory_frozen`，若实现）

**FR-08** 在**首次建立该方向持仓**的 tick：

- 使用开仓价及开仓前后若干根 K 线，拟合一条平滑价格轨迹（具体阶次、坐标系由设计文档定义，建议支持对数价格以无量纲化）。  
- 拟合需满足有效性约束（例如：存在理论极值点、极值点在将来时刻、曲率方向与持仓方向一致等）；不满足则本 tick 不启用轨迹保护，可降级为 hold 或仅依赖全局风控。

**FR-09** 拟合成功后，将锚点参数序列化保存；后续 tick 只读取锚点 + 当前行情，**禁止**因新 K 线而重新拟合替换锚点。

**FR-10** 每个 tick 计算：

- 预测价 `P_pred`（由冻结参数与持仓 bar 数推导，可对高阶项做时间衰减，但锚点参数本身不变）；  
- 实际价 `P_act`；  
- 残差（建议无量纲，如 `ln(P_pred / P_act)`）；  
- 贴合度 `fit ∈ [0,1]`，随 `|残差|` 增大而单调下降，且对正负残差对称；  
- 偏离方向枚举：`aligned` | `favorable` | `adverse`（文案映射见 6.6）。

**FR-11** 容忍带 `tol`：

- 由**手续费率**与**开仓附近局部波动**共同决定，取二者推导下限的较大值，不新增用户可调 GA 参数；  
- 保护单触发价相对当前价的距离不得超过产品规定的**最大相对距离**（建议默认 5%，可配置常量，非 per-user 参数）。

**FR-12** 利润门槛：若按模型计算的「理论最优退出价」扣除双边手续费后仍无法保证正收益，则**不得**挂轨迹类止盈保护单，仅允许 hold 或等待硬止损/全局风控。

**FR-13** 硬止损：当不利偏离超过 `max(2 × tol, 最大相对距离对应的对数阈值)` 时，输出市价全平意图。

**FR-14** 有利偏离时挂止盈/保护单：触发价不得被 raw 残差无限推远，须受 `tol` 与最大相对距离双重约束。

**FR-15** 轨迹类模型的所有平仓输出均为**全仓**；不支持分批减仓（若未来支持，须另开 PRD 修订）。

### 6.4 默认模型（`baseline`）

**FR-16** `baseline` 的行为由承接方在项目内单独编写《baseline 平仓说明》，但须满足：

- 与 FR-05 优先级框架兼容；  
- 切换为 `trajectory_frozen` 时，`baseline` 的单元测试基线不得被破坏（回归用例保留）。

### 6.5 执行与对账（策略层边界）

**FR-17** 策略层只输出语义化意图，例如：`hold` | `upsert_protection` | `market_close_all`，附带触发价、方向、幂等键等字段；不直接调用交易所。

**FR-18** 执行服务负责将意图转为实际订单；策略层不得根据「是否回测」分支采用不同平仓公式。

**FR-19** 保护单类型须区分：

- 止盈类：触发价位于对持仓**有利**一侧；  
- 保护止损类：触发价位于对持仓**不利**一侧。  

回测引擎须按第 8 节规则判断成交。

### 6.6 用户界面

**FR-20** 配置页：平仓模型下拉，选项为可读中文/英文文案，提交值为内部枚举。

**FR-21** 监控页（仅当当前模型为轨迹类时展示扩展字段）：

| 展示名（示例） | 含义 |
|----------------|------|
| 轨迹贴合度 | `fit`，百分比或 0–100 |
| 偏离方向 | 贴合 / 有利突破 / 不利突破 |
| 理论极值价 | 冻结轨迹上的参考极值价格 |
| 当前保护价 | 已下发或计划下发的保护触发价 |

**FR-22** 禁止在面向用户的文案中出现未释义的单字母变量、内部 JSON 字段名或裸公式。

**FR-23** 当扩展字段无数据时，界面降级展示，不得报错或白屏。

## 7. 数据与接口（逻辑契约）

以下为逻辑字段，实现方可映射到数据库列、消息体或缓存键。

### 7.1 运行配置

| 字段 | 类型 | 说明 |
|------|------|------|
| `exit_model` | enum string | `baseline` \| `trajectory_frozen`，默认 `baseline` |

持久化位置建议：进化/回测任务表、实盘实例表、冠军/参数包快照（若有）。

### 7.2 每 Tick 策略输出（每个持仓）

| 字段 | 类型 | 说明 |
|------|------|------|
| `action` | enum | `hold` \| `upsert_protection` \| `market_close_all` |
| `protection_side` | enum | `take_profit` \| `stop_loss`（仅 upsert 时） |
| `trigger_price` | decimal | 保护单触发价 |
| `quantity` | decimal | 全平时=当前总持仓；保护单=全仓数量 |
| `reason_code` | string | 审计用，如 `hard_stop_adverse`, `global_max_hold` |

### 7.3 轨迹类中间状态（持久化）

| 字段 | 类型 | 说明 |
|------|------|------|
| `anchor_frozen_at` | timestamp | 锚点冻结时刻 |
| `anchor_payload` | JSON | 冻结参数、拟合窗口元数据 |
| `last_fit` | float | 上一 tick 贴合度 |
| `last_direction` | enum | 上一 tick 偏离方向 |

**约束**：`baseline` 模型不得读写 `anchor_payload`；两种模型使用独立状态槽，避免互相覆盖。

## 8. 回测成交规则（产品级）

对**空头**持仓（其他方向需镜像定义）：

| 保护单类型 | 触发价相对现价 | 当根 K 线判定成交的价格 |
|------------|----------------|-------------------------|
| 止盈 | 低于现价 | 若最低价 ≤ 触发价，则视为成交 |
| 保护止损 | 高于现价 | 若最高价 ≥ 触发价，则视为成交 |

市价全平：按项目既有市价成交规则（如本根收盘价、下一根开盘价或滑点模型），须在《回测规范》中写清，且与实盘「尽力成交」语义对齐说明。

## 9. 非功能需求

| 编号 | 要求 |
|------|------|
| NFR-01 | 单次 tick 平仓计算在常规硬件上应完成于策略 tick 预算的 10% 以内 |
| NFR-02 | 所有平仓决策可记录输入快照（行情、锚点、配置）供事后重放 |
| NFR-03 | 配置变更仅对**变更后新开仓**生效，或明确定义「是否重算锚点」策略并在界面提示 |
| NFR-04 | 国际化：固定 UI 文案走 i18n 键，接口返回的枚举可由前端映射 |

## 10. 风险与假设

**假设**

- 执行层能可靠地下发、撤销、查询保护单。  
- 行情源在时间上与决策 tick 对齐，无系统性前视。  
- 手续费率在决策时可获知（或保守上界已知）。

**风险**

| 风险 | 缓解 |
|------|------|
| 拟合失败导致无保护 | 降级为 hold + 全局风控；监控拟合失败率 |
| 容忍带过窄导致频繁止损 | `tol` 取下限较大者；硬止损使用 2×tol |
| 容忍带过宽导致挂单过远 | 最大相对距离硬顶 |
| 回测与实盘保护单行为不一致 | 共用决策函数 + 独立《执行层差异说明》 |

## 11. 验收标准

### 11.1 单元测试

- [ ] 轨迹拟合：构造已知系数序列，恢复参数误差在约定阈值内；无效形态被拒绝。  
- [ ] `tol` 在「仅手续费」与「仅波动」两种极端输入下取较大者。  
- [ ] `fit` 对正负残差对称；方向枚举与残差符号、持仓方向一致。  
- [ ] 硬止损仅在不利偏离超阈时触发。

### 11.2 策略集成

- [ ] `exit_model=baseline` 时，与改造前 golden case 输出一致。  
- [ ] `trajectory_frozen`：首次建仓冻结锚点；模拟加仓后锚点 JSON 不变。  
- [ ] 任意模型下平仓数量均为全仓。

### 11.3 回测

- [ ] 空头止盈：触发价 below → 用 low 触发。  
- [ ] 空头保护止损：触发价 above → 用 high 触发。

### 11.4 API 与 UI

- [ ] 创建任务/实例可读写 `exit_model`；缺省为 `baseline`。  
- [ ] 监控页在两种模型下均可加载；轨迹字段缺失时不崩溃。
